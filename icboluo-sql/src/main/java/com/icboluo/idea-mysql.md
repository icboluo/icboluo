## 索引失效

当作为状态值的时候，如果该值的出现的比率较高，则索引失效

    首先进入该值的节点，然后进入下一个节点，这是用索引
    进入下一个节点（虽然节点数多，但是只走树的一个分支，很可能比较快），这是用全局扫描

group by 和order by的时候，很容易造成效率低下，可以对count语句进行优化

联合索引的效率和普通索引的效率一致
